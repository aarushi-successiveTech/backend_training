# Introducton to express
- It is a minimal and flexible Node.js web application framework that provides a list of features for building web and mobile applications easily.
- It simplifies the development of server-side applications by offering an easy-to-use API for routing, middleware, and HTTP utilities.
- Uses a series of middleware functions to process requests — giving you full control over the request-response cycle.
- Express simplifies defining routes and HTTP methods (GET, POST, etc.) to handle incoming requests.
- Ideal for building RESTful APIs and structuring apps using MVC (Model-View-Controller) patterns.
- It is also called de facto standard server framework for nodejs. 

## Why Express? 
- Used in web development to simplify and accelerate the process of building web applications, particularly those built on Node.js.
- Acts as a layer on top of Node.js, providing a more organized and efficient way to handle routing, middleware, and HTTP requests.
- Makes it easy to define routes and handle different HTTP methods (GET, POST, etc.) for your application.

## Applications 
1. Real-Time Apps – Used with Socket.io for chat apps, notifications, etc
2. Single Page Applications (SPAs) – Serve SPAs like React, Angular, or Vue
3. Microservices – A great choice for building small, scalable services

# REST API 
- Representational State Transfer Application Programming Interface is a way for different software systems to communicate with each other over the internet.
- It uses a set of architectural constraints to define how clients and servers interact, primarily relying on standard HTTP methods.
- Key components of a REST API include resources, HTTP methods, URLs, requests, responses, and headers. 
- components of REST API : 
- 1. Endpoint - specifies the resource the client wants to access or manipulate. It's essentially the address of the resource on the server. 
- 2. HTTP methods - methods, such as GET, POST, PUT, and DELETE, define the action the client wants to perform on the resource. 
- 3. Request Headers - provides additional information about the request, such as the content type, authentication details, and other metadata. 
- 4. Request Body - contains the data that the client sends to the server, often used for creating or updating resources. 
- 5. Resource - any data or content that can be accessed or manipulated via the API, like text, images, or documents. 
- 6. Client - application or software that initiates the API request to access or modify resources. 
- 7. Server - hosts the resources and processes the client's requests. 
- 8. Response Structure - The server sends back a response containing the requested data or an indication of the success or failure of the request. This response also includes status codes and potentially headers.

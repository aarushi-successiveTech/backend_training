# Client-server Model
- It is a distributed application architecture that divides tasks or workloads between servers (providers of resources or services) and clients (requesters of those services)
- Here, a client sends a request to a server for data, which is typically processed on the server side. The server then returns the requested data to the client.

# How client-server model works 
1. client sends request to the server over the network 
2. server receives the client's request
3. server processes the request which may involve running logic or accessing resources
4. server sends back response to client 
5. client receives the response and displays/processes the information to user 
6. this exchange happens over a communication protocol (HTTP/HTTPS)

# Advantages
- Centralized Data Management
- Cost Efficiency
- Scalability
- Data Recovery
- Security 

# Real world example 
- Email Systems
- - Client: The user’s email client 
- - Server: The email server 
- - working: The email client requests emails from the server, and the server delivers them. Similarly, when the user sends an email, the client communicates with the server to send the message.

# Backend Architecture 
- It refers to the structure and organization of the server-side components of a software application.
- It encompasses how the server, application, and database interact to manage data, handle requests, and ensure the application functions correctly.
- Thus, it's the blueprint for how the backend works, including how different elements communicate and process information. 

## Need of Backend Architecture 
- scalability 
- security 
- performance 
- mainainability 

### Types of Architectures 
1. Monolith Architecture : 
- - It is a single-tier architecture
- - The size is large as all the components are tightly coupled as single, indivisible unit
- - Deployed as a single Unit 
- - Commuinication between the component is faster
- - Limited technology choices are available 
- - Less flexible as components are tightly coupled 

2. Microservices Architecture : 
- - It is a multi-tier architecture 
- - The size is small as all components are loosely coupled 
- - Individual services are deployed independently 
- - Communication between componenets is slower due to network calls 
- - Freedom to choose the best and suitable technology for each service
- - Highly fexible as components can be developed, deployed and scaled independently
